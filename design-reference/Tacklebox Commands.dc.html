Command Reference

Every Tacklebox command, grouped by feature, with separate walkthroughs for players and officers. Commands run as Discord slash commands — type / in any channel the bot can see.

Party Manager

/party

Connects a Google Sheet to Discord for party signups. Officers start a party from a sheet URL, players sign up for weapons/roles via buttons, and officers assign roles in the sheet and sync them back to Discord.

COMMANDS
/party start <url>Start a new party signup from a Google Sheet URL. Cap of 10 active parties per server; parties auto-expire after 10 days.
PLAYER FLOW
  1. Click Sign Up, select the weapon(s) you can play, add notes if needed (low spec, early leave). Written to the sheet automatically.
  2. Click Sign Up again any time to update your selections.
  3. Click Un-Sign Up to remove your signup from the sheet and party.
  4. Click View Loadout after officers sync to see your assigned weapon and required equipment.
OFFICER FLOW
  1. Run /party start <sheet URL>. Pick a tab if the sheet has multiple.
  2. Click Add Details for event title, UTC time, meeting place, gear, and description.
  3. Copy signups from Column A into Column F next to their assigned weapon (names must match exactly).
  4. Click Sync to pull Column F into the embed. Repeat as your roster changes.
  5. Click End Party to close signups and remove buttons. Sync/Details/End can be used by the starter or anyone with Manage Server.

Copy the sheet template (File → Make a copy) and share it with the bot's service account — tacklebox-bot@tacklebox-489703.iam.gserviceaccount.com — as Editor.

A–C — Player signups, written automatically. Don't edit.

D — Role (Tank, Healer, DPS...). E — Weapon name, one per row.

F — Assignments. Paste player names exactly as they appear in Column A.

G+ — Equipment loadout; header row names the slot, cells list required items. Shown to players via View Loadout.

⚠️ Don't rearrange or delete columns A–F — the bot reads fixed column positions.

Loot Split

/loot

Splits loot silver evenly among party members. Start the split, members react to join, officers can adjust the player list, and the bot calculates each player's cut after repairs and guild tax.

COMMANDS
/loot startStart a new loot split message in the current channel. Optional: voice_channel for the Ping button, tax_rate to override the guild default.
/admin set_default_taxAdmin only. Sets the guild's default tax rate (0–100%). Default is 35% if never set.
PLAYER FLOW
  1. React to the loot split embed with any emoji to join — one reaction counts, doesn't matter which emoji.
  2. If an officer excludes you, you won't appear on the payout checklist even if you reacted.
  3. If self-marking is enabled, click Mark Myself Paid once you're paid. Only works if you're unpaid and included.
OFFICER FLOW
  1. Run /loot start. The bot posts an embed; unique reactors are counted automatically.
  2. Click Ping Voice Channel to @mention everyone in the selected voice channel.
  3. Click Player Override (initiator only) to Add, Exclude, or Reset players — up to 40 per list, deduplicated against reactions.
  4. Click Calculate and fill in Total Silver, Repair Cost, and Tax Rate (pre-filled with the guild default).
  5. Run /admin set_default_tax <0–100> to change the default used by Calculate.

After Calculate runs, the bot posts a checklist with three states: ❌ unpaid, ✅ paid directly, 💰 deposited into bank.

Marking paid — dropdowns below the checklist show only unpaid players; picking one marks them ✅. More than 25 unpaid adds more dropdowns.

Bank — opens a private panel to deposit unpaid players' cuts into /bank; Bank All covers everyone unpaid.

Reset — sends ✅/💰 players back to ❌; banked players have the split withdrawn from their balance.

⚠️ The checklist only responds for 1 hour after posting. Bank changes already made stay in /bank — use /bank update for manual corrections.

Regear Timer

/regear

Tracks gear reimbursements. Officers start a timer for a player, the player reacts to confirm they've collected their gear, and the bot notifies the officer to empty the chest — or marks it expired if time runs out.

COMMANDS
/regear start <user> <chest>e.g. /regear start @Angler 3. Each player can have up to 5 active timers.
/admin set_reminder <hours>Admin only. 1–168h, must be less than expiry. Default: 24h.
/admin set_expiry <hours>Admin only. 1–168h, must be greater than reminder. Default: 48h.
PLAYER FLOW
  1. The bot tags you with a message showing the chest number and how long you have to claim.
  2. React with ✅ once you've picked up your gear — on the original message or a reminder, either works.
  3. If you haven't reacted after the reminder window (default 24h), a follow-up reminder is sent.
  4. If the timer expires before you react, it's marked expired — reach out to the officer directly.
OFFICER FLOW
  1. Run /regear start <@player> <chest>. The bot posts publicly and auto-adds a ✅ reaction.
  2. If unclaimed after the reminder threshold, the bot sends a follow-up tagging the player again.
  3. On claim, the bot edits the message to "✅ Regear Claimed" and tags you to empty the chest.
  4. On expiry, the bot marks it "❌ Regear Expired" and tags you to empty the chest anyway.

World Clock

/clock

Creates and manages voice channels that display the current time in different time zones, updating every minute.

Administrator permission required for every /clock command.

COMMANDS
/clock setupCreates 3 default clocks — UTC, Eastern, Pacific. Running again deletes and replaces existing clocks (confirmation required).
/clock addOpens a modal for a label and IANA time zone string (e.g. Europe/London).
/clock replacePick a clock, then enter a new label and time zone.
/clock removePick a clock from the dropdown to delete its voice channel.
/clock listLists all configured clocks and their time zones.

Bank

/bank

Tracks player silver balances per server. Balances can go negative. All responses are visible only to whoever ran the command.

COMMANDS
/bank_view <user>Shows a player's current silver balance.
/bank updateAdmin only. Adjusts a player's balance by adding or subtracting silver.
/bank payout <user>Admin only. Resets a player's balance to zero.
/bank debt_listAdmin only. Shows all recorded balances, highest to lowest.

Admin

/admin

Guild-wide configuration. All commands require the Administrator permission and are run once per server; responses are ephemeral.

Changes to reminder/expiry only apply to timers started after the change — running timers keep their original settings.
COMMANDS
/admin set_default_tax <0–100>Sets the default tax rate for loot splits — pre-fills the Calculate button. e.g. 35 means 35%.
/admin set_reminder <hours>1–168h, must be less than expiry. Default: 24h.
/admin set_expiry <hours>1–168h, must be greater than reminder. Default: 48h.

Permissions

By default, most commands are open to any server member. Discord's own Integrations settings let admins restrict or grant access per command, by role, user, or channel — on top of whatever the bot itself checks.

Getting there

Server Settings → Integrations → find Tacklebox → Manage. You'll see every registered command with its own toggle — default is all members allowed.

RESTRICT /party TO OFFICERS
  1. Find party in the bot's command list.
  2. Toggle off @everyone.
  3. Add roles or members → select your officer role.
  4. Toggle that role on, then save.
EXPOSE /admin TO A CO-ADMIN ROLE
  1. Find admin in the bot's command list.
  2. Toggle off @everyone, add your trusted non-Administrator role.
  3. The bot still rejects /admin from anyone lacking real Administrator permission — Discord toggles and bot checks are independent layers.
